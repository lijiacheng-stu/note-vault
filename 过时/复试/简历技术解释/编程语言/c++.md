#### 二、面向对象的核心概念：
##### 1.类和对象
声明、定义、初始化、赋值。
声明是告诉编译器某个变量或函数的存在，但并不分配内存或提供具体实现
定义是对于变量，定义会分配存储空间；对于函数，定义会提供函数体。
初始化是在定义变量时为其赋予一个初始值。当定义时未初始化，则其值不确定的情况。也就是说定义之后一定会伴随初始化。
赋值是为已经定义的变量赋予一个新值。赋值操作发生在变量定义之后。

##### 2. 封装：
核心思想：数据（属性）和行为（方法）捆绑在一起，并对外隐藏内部实现的细节，只暴露必要的接口。
通过引入类和对数据和行为进行访问控制（public,private,protected）实现。
%%
解释：
- 数据和行为捆绑在一起： 
将数据（属性）和操作这些数据的方法（行为）组合在一个单元（通常是类）中，而不是将它们分散在程序的各个部分。
%%
#### 3.继承
核心思想：一个类可以基于另外一个类来创建，继承父类的属性和方法，并可以扩展和修改它们。
作用：代码复用，扩展功能，层次化设计
在c++中继承与java中不同，以B继承A为例子，如下：
```
class A{};
class B::public A{};
```
这里的public的作用的对A中的权限进行调控。public下，public为public，protected为protected，private为不可访问。
protected下，public protected为protected，private为不可访问。
#### 4.多态
核心思想：同一个接口可以表现出不同的行为，具体行为取决于实际的对象类型。
在c++中，解决的问题是函数名相同的情况下，调用哪儿个函数，父类还是子类的？父（子）类的话，调用哪儿个具体的。

**函数调用机制**：静态绑定和动态绑定。

#### 四、内存管理：
```
class A{}
int a; //全局存储区
int main(){
	int b; //栈
	const char* c = "hello world!"; //c在栈中，helloworld在常量存储区
	int* d = new int; //d指向的内存在堆中
	A a;
	A* b = new A;
}
```
c++内存区域分为5个区域。堆、栈、全局/静态存储区，常量存储区，代码区,如上。这也延申出访问变量的方式有三种，直接访问、指针、引用。
对于对象的存储，与java相比，c++的内存模型更加丰富。c++中的对象和普通的变量没有区别，堆栈全局存储区都可以存储，有上述三种访问。而java对象存储在堆中，通过指针访问。
