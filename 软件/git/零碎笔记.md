一、get started
版本控制系统VCS包括LVCS,CVCS,DVCS。git就是一种分布式版本管理系统（DVCS）。
git将数据看成是一个微型文件系统的一系列快照。在每一次commit后，会保存系统的状态，也就是拍一张快照，并保存这张快照的引用（reference）。
一个Git由三个主要状态modified,staged和committed。对应三个主要的部分：working tree（是各个版本中的一个,staging area（是一个文件，标记commit后进入下一个版本的文件）,git diretory（保存项目各个版本的snapshot，是.git文件夹）。工作流是在working tree中编辑，选中需要放入下一个版本的修改标记staged，然后commit。
对git的配置文件在三个文件夹中，git安装目录下的管理所有用户system，某用户目录下的管理这个用户global，项目目录管理这个项目local。key相同，值不同，则后一个覆盖前一个。配置项：名字，邮箱，文本编辑器，分支名称。
查：
```
git config --list //列出所有的键值对
git config user.name //查某个键的值
git config --show-origin rerere.autoUpdate//列出生效值得配置文件目录和值

```
改：
```
git config --global user.name "John Doe"  git config --global user.email johndoe@example.com
```

帮助系统记住两个：
```
git help config //打开manpages中得config内容
git config -h //精简
```
