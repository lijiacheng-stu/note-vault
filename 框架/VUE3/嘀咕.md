怎么就点击一个按钮，就会劈里啪啦地显示评论呢？
评论是一个组件，由属性v-show控制。当点击按钮后，引起了点击事件，从而让v-show变成了true，从了引起了评论组件的生命周期。生命周期的created的阶段会执行setup中的方法，其中有一个方法就要做fetchComments，作用是根据postId在API中获得对应的comments，并放在一个响应式的数组中。然后是生命周期的render阶段，根据这个相应式数据，用v-for属性，完成组件的加载工作。

router的作用的什么？
router就是一个映射表，A给路径，通过对照表router，就呈现对应的组件B。
这会引申出一些问题，
怎么给路径呢？
- `<RouterLink to="/">Home</RouterLink>`
- 用(点击等)事件：
```typescript
const onRouterLinkClick = () => {
	const router = useRouter();
	router.push({
		name: "user",
		params:{
			userId: props.userId
		}
	})
}
```
哪儿里呈现组件B呢？
- router得应用于一个应用`const app = createApp(App); app.use(router);` 此时，route中的每一项的出口都得在App的`<RouterView></RouterView>`中，而router如果由子route，则变化的就是跳转到对应route后`<RouterView>`相应位置，这个哲学时始终符合的。
如果说B组件会因为信息不同而呈现不同的内容呢？
- A将索引内容放在路径中，这些路径都会指向组件B，B通过查看路径，得到数据后，再呈现。这也就是动态路由了。怎么查看数据呢`const route = useRoute(); const data = route.params.useId`
