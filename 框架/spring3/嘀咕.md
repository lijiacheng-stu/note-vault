怎么理解JdbcTemplate、Jdbc Driver、DataSource、JDBC、H2、mysql、HiKariCP之间的关系？
答：
- 答：JDBC是接口，Jdbc Driver是这个接口的实现，通过依赖导入可以实现驱动的导入
有了导入驱动的JDBC理论上就可以实现对数据库的访问了，但是简单的操作需要连接数据库、处理异常、写sql等繁琐的操作，因此为了简化操作就出现了对JDBC的封装类JdbcTemplate。JdbcTemplate能够让我们只关注sql部分，连接数据库和处理异常等不再需要我们关心。
H2和mysql都是关系型数据库，但是H2是in-memory数据库而mysql是存储在硬盘的数据库。H2因为不太适合生产场景，但是在测试的时候非常好。java使用数据库必须先建立连接，如果每次操作都建立和释放连接不合适，因此有连接池connection pool，需要sql操作时从池中拿数据。值得说明的是DataSource是连接池的接口，而HikariCP是它的实现类,且spring默认使用HikariCP，一个连接池的创建只需要uri,username,password即可（uri定位到哪儿台电脑的哪儿个具体的库，（username,password）来获得权限）。JdbcTemplate的使用是依赖连接池的。如果显式地定义JdbcTemplate实例并放在application context中，是这样地：
```java
@Configuration
public class Config {
    @Bean
    public DataSource dataSource() {
        // 假设这是连接 MySQL
        HikariDataSource ds = new HikariDataSource();
        ds.setJdbcUrl("jdbc:mysql://localhost:3306/mydb");
        ds.setUsername("root");
        ds.setPassword("123456");
        return ds;
    }

    @Bean
    public JdbcTemplate jdbcTemplate(DataSource dataSource) {
        return new JdbcTemplate(dataSource);
    }
}

```
隐式地呢？
如果引入H2，springboot会自动生成H2地实例，和JdbcTemplate地实例，并且让它们wired起来。之后自动执行schema.sql中的语句创建包，data.sql中的语句加入数据。
如果是mysql可能就不会了。spring.datasource.initialization-mode=always需要它指明。
