#### 一、变量
##### 1.变量的内容为函数的认识
三个方面的认识：
- 变量的内容可以是函数
- 函数的参数的类型可以指定，函数也可以作为一个参数
- 内容为函数的变量可以当作正常的函数名进行调用
```
let x = function sum(a:Function，b:number) {
            a()
            console.log(b)
        }
x(()=>{console.log(15)},47)
```
在以上的函数说明了这一点。x的变量内容是sum函数。sum函数中的参数类型被指定，a指定为Function，b指定为number。x的调用体现了第三条。

#### 四、简写

##### 2.函数的简写
- 箭头函数
```
(形参1,...,形参n)=>{
	函数体
}
```
个人：用于给函数的形参赋值的情形，形参的赋值不需要定义函数的名称，可以略掉

##### 3.文件路径
- index.ts文件在被引用时的简写：
```
文件的目录结构：
src
	components
	types
		index.ts
```
types文件夹用于定义自定义的规范，如接口。有个特别的文件index.ts，被引用时`@/types/index.ts`可以简写为`@/types`

#### 其他碎片化知识点
##### 1.文件路径中的@
当person.vue中的ts要引入index.ts中定义的接口时，`src/types/index.ts`可以写成'@/types'，仿佛@就是直接表示为了'src/'
```
文件的目录结构：
src
	components
		Person.vue
	types
		index.ts
App.vue
```
##### 2.类型的等价写法
- `Array<person>`和`person[]` 在表示类型时等价，如：
```
import{type person} from '@/types';
let s2:Array<person>=[
        {name:'zhangsan',age:18,address:'宁都'},
        {name:'lisi',age:20,address:'宁都'}
    ]
与
let s2:person[]=[
        {name:'zhangsan',age:18,address:'宁都'},
        {name:'lisi',age:20,address:'宁都'}
    ]
```
