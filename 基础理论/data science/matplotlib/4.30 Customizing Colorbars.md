图片的理论：
1. 对RGBA中A(alpha，透明度)的认识：
含义：前景和背景的重叠部分进行混合色彩时，前景的权重
```
 fg:前景
 bg:背景
 前景和背景重叠部分的rgb：fg * alpha + bg * (1 - alpha)
```

2. 对灰度图的认识：
灰度：0~255
含义：rgb的数值相同时，呈现出来的图像是灰色的。当0是纯黑，255是纯白。

3. 彩色图转灰度图：
原理：将像素点的色彩转化成对应的亮度，效果是彩色图转灰度图。由于人们对红绿蓝的颜色的敏感度不同 因此在混合时每种颜色的权重不同`RGB_weight = [0.299,0.587,0.114]` 

色图理论：
各种colormap色图：
- Sequential colormap 顺序色图 
- Divegent colormap 发散色图
- qualitative colormap 定性色图
彩色打印转成灰度打印的时候，顺序色图能够保持数据越大亮度越大。而定性色图是无序的，可能亮度高的数据并不是数据大的。而且就算是不灰度打印，我们也会被一些亮度高的彩色吸引注意力。所以表示连续数字的时候要用顺序色图而不是定性色图。

如果一个对象的类实现了 **`__call__`** 方法，这个对象就变得可调用，意味着你可以像调用函数一样调用它，并传递参数。```

```
class CallableObject:
    def __init__(self, value):
        self.value = value

    def __call__(self, x):
        return self.value * x

# 创建一个对象
obj = CallableObject(10)

# 调用对象并传递参数
result = obj(5)  # 实际上调用了 obj.__call__(5)

print(result)  # 输出 50
```
