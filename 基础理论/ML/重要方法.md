		- 意义：measures the ratio of the variance that is explained by the model.模型能够解释的方差占总方差的比例。

		- 怎么理解：

				- 真正例：这个预测为正的样例，结果真的为正。正确地预测为是目标类别。

				- 第二个字母反映预测的情况
			-  $\displaystyle accuracy = \frac{TP + TN}{TP + TN + FP + FN}$
			- 意义：正确预测的数量占所有预测的比例。
			- 缺陷：若比例55开，只预测正确，也有50%的accuracy。失衡的话，可以更高。即，当 **类别不平衡（class imbalance）** 时，accuracy 可能 **误导模型效果**。

## 评价指标
四类评价指标：
- during each epoch, use batch in training set:
	- 记录每个batch的loss，最后加和 / m，得到平均loss，判断模型的波动情况
- at end of each epoch, use training set and validation set：
	- 用训练集和验证集测试性能，判断是否过拟合。
- at end of training using training set, use validation set：
	- 用验证集测试性能，用于评估当前训练的模型的性能，选超参数和模型。
- end of training using training set + validation set, use test set to measure the ability of model:
	- 用测试集，评估模型性能和泛化能力

## 怎么看待一个模型的实例
一个模型类中，往往至少包含两个方法，一个是构造器，一个是forward
从构造器角度：
- 模型实例是仅包含参数和模块的容器，且模块和参数都能通过属性获得。属性 是键，模块和参数是值。模块和参数之间是组合关系。
从构造器+forward角度：
- 模型是由若干子模块组合成的有机成立，包含了 子模块以及他们直接的数据流动。forward之间是按属性连接的，因此，可以替换模块，而不需要改forward代码。

## 如何准备数据
- 准备原始数据
- 对原始数据进行划分，用于构建各自的training set，validation set， test set
	- 因为对原始数据可能要进行预处理操作，有些预处理操作可能是需要通过training set学习的。
- 构建各自的数据集
- 分别对三套数据集构建DataLoader，得到train_loader, valid_loader, test_loader



- 从high level来看，输入的都是形状都是(m, the shape of instance in X_train)，输出都是(m, the shape of instance in y_train)
- 模型本质上是“单样本函数”， 每次输入a instance in X_train,输出一个y_target; 但是可以并行处理多个，因为他们instance之间没有同步关系，可以并行计算。batch是“向量化执行”，并行执行多个样本。
- 所以，在考虑模型结构的时候，仅仅考虑单样本即可。
- 构建参数形成的模型的时候，应该考虑batch，但是也是逐个样本，进行循环，结果放在一个向量里面。
