# 1. 什么是Feature Pyramid Networks？
## 1.1 引言
### 1.1.1 挑战
识别不同尺度上的物体是计算机视觉中的一个基本挑战。
### 1.1.2 解决方案
- 方法一：Featurized image pyramids。
Featurized image pyramids是基于图像金字塔的特征金字塔，一个物体的尺度变化可以通过在金字塔中改变其层级来抵消。小尺度物体的识别，可以通过扫描更高分辨率的金字塔层级来实现；大尺度物体的识别，则扫描更低分辨率的金字塔层级。
优点：所有的金字塔级别语义都没强，包括高分辨率的级别
缺点：推理时间长，训练所需的显存很大。
- 方法二：Single feature map。能力
卷积网络能够表示高级语义的同时，对目标的尺度变化具有鲁棒性，只需单个特征图即可实现目标检测。
注：卷积网络提升了单一输入尺度下识别能力。当时最先进的方法是结合方法一和方法二
- 方法三：Pyramidal feature hierarchy
卷积网络在前向传播过程中，会产生特征层次结构，这个结构有固有的多尺度，金字塔的形状。Single Shot Detector（SSD）首次把这个特征层次结构以特征化图像金字塔来看待，企图使用不同层中的多尺度特征图的同时，不引入其他计算和存储开销。
缺点：不同层之间尽管有不同的空间分辨率，但是有很大的语义鸿沟。高分辨率的图仅有低级的特征，这些特征的表达能力不足以用于目标检测。一种可行的办法是，增加卷积网络的深度，从高层构建特征金字塔。

![[Pasted image 20260529215759.png]]
目标：本文的目标是利用卷积网络的特征层次结构，构建在所有尺度上都具有强大语义的特征金字塔。 （对方法三的改进）
## 1.2 特征金字塔网络Feature Pyramid Networks
![[Pasted image 20260530155425.png]]
- 自下而上的路径
	- 自下而上的路径是骨干卷积网络的前馈计算。
	- 阶段(stage)：常常有许多层产生相同大小的输出地图，我们称这些层处于同一网络阶段。
	- 层级（level）：为每个阶段定义一个金字塔层级，用于构建特征金字塔。每个阶段的最后一层的输出作为这个阶段的金字塔层级。
	- ![[Pasted image 20260530155735.png]]
	- （以ResNet为例，这两个蓝色残差单元输出的特征图大小相同，所以它们处于相同的阶段。将最上面这个蓝色的残差单元的输出，作为一个金字塔层级。最终的结果是相邻金字塔层级， 分辨率存在2倍关系。）
- 自上而下的路径 和 横向连接
	- 思想：
		- 上一层：语义强，分辨率低
		- 下一层：语义弱，分辨率高
		- 上一层 融合 下一层： 语义强，分辨率高的特征图。
	- 融合的困难：上下两层分辨率不匹配，特征图的通道数量不匹配。
	- 自上而下的路径：解决分辨率不匹配的问题。通过最近邻上采样，将空间分辨率提升2倍，得到上采样的特征图。
	- 横向连接：解决特征图的通道数不匹配问题。通过`1*1`卷积，得到通道变化的特征图。
	- 最终的特征金字塔层级：上采样的特征图 和 通道变化的特征图逐元素相加，然后再进行一次`3*3`卷积操作，得到最终的特征图。
#  2. Feature Pyramid Networks是怎么应用的？

## 2.1 什么是Fast R-CNN？
![[Pasted image 20260530171438.png]]
- 输入：一张图片，以及一组候选区域（RoI）的边界框
	- 每个边界框由4个数值表示，用来描述这个RoI在图片中的范围
	- 候选区域是目标可能存在的区域，但是具体的目标类别未知
	- 候选区域的生成依赖的是其他的算法，而非Fast R-CNN
- 将一张图像输入深度卷积网络，得到一个特征图
- 将候选区域的边界框，按特征图相对原图的空间缩放比例，映射到特征图。 
- 对特征图上的每个的RoI进行这样的操作：
	- 使用 RoI Pooling Layer，将不同尺寸的 RoI 池化为固定大小的特征表示，以便输入后续全连接层（FC），得到RoI特征向量。
	- RoI特征向量输入到两个输出头中，一个头用于输出各个类别的概率，一个头用于输出边界框坐标的偏移量，用于对坐标进行修正。
## 2.2 如何将FPN应用于R-CNN？
分析：
- 上述的Fast R-CNN，预测头在单尺度特征图上逐个对候选区域进行预测。
- 候选区域有大有小
思想：
- 对图像使用FPN，得到特征金字塔。将不同尺度的候选区域分配到金字塔的各个层级：大的候选区域是粗粒度对象，分配给给金字塔的高层，小的候选区域是细粒度对象，分配给金字塔底层。用相同的预测头用于各个层级。

## 2.3 什么是RPN？
- 将一张图像输入深度卷积网络，得到一个特征图
- 将一个小网络在卷积特征图上滑动：
	- 小网络每次以特征图的`3*3`的空间窗口作为输入，输出256-d的低维特征（256，`3*3`卷积实现; ）
	- 256-d的低维特征输入到两个输出层cls和reg，用于预测k个边界框。因此，reg层输出k个由4个数值表示的边界框，cls层输出k个由两个数值表示的object score，用于计算每个变框存在目标的概率。（2k和4k，`1*1`卷积实现）
![[Pasted image 20260530200326.png]]
锚的概念：
上面提到的k个边界框的数值是相对于k个预设的锚而言的。一个锚是特殊的边界框，以滑动窗口为中心，并与一个尺度（scale）和宽高比（aspect ratio）相关联。（文中有三个3个尺度和3个宽高比，所以每个滑动位置产生k=9个锚）。
- 现在这个滑动位置输出了k个4个数值表示的边界框，第一个边界框的四个值是相对于第一个锚而言，tx、ty中心偏离了多少，tw、th宽高偏离了多少。
![[Pasted image 20260530202646.png]]

## 2.4 如何将FPN应用于RPN？
分析：
- 上述的RPN，小网络在单尺度特征图上的一个滑动位置，预测相对于3个不同尺度的锚的边界框。
思想：
- 对图像使用FPN，得到特征金字塔。每层相对一种锚尺度进行预测。相对于尺度小的锚的边界框预测，在特征金字塔底层上进行。相对于尺度大的锚的边界框预测，在特征金字塔高层上进行。每层用于预测的小网络是相同的。
- 所以在某层上，单个滑动位置预测3个边界框，不是9个了，对应3个比例。

- 用selective search算法得到2000个region proposals
- 对region proposal进行wrap，使图片缩放到适应CNN网络需要的尺寸
- 用CNN对warped region提取一维度特征，用特定类别的SVM对特征进行打分
	- 用CNN得到的是特征图，后面怎么跟FCs的？难道后面经过逐通道的平均池化吗？不，用的是展平，现在空间维度上展，后在通道维度上
- 用greedy non-maximum suppression，对region proposals进行筛选
## Training
### (1) Supervised pre-training
- 用辅助训练集((ILSVRC2012 classification),对CNN进行预训练
- 目的：预训练骨干CNN 
### (2)Domain-specific fine-tuning
- 用 (N + 1)-way classification layer替换CNN’s ImageNet-specific 1000-way classification layer
- 任务：**N+1 类多分类**
- 数据集：用warped proposal windows作为input，**IoU≥0.5 为正样本**，其余视作背景负样本
- mini-batch：32 positive windows + 96 background windows
- 目的：微调骨干CNN
### (3) Object category classifiers
- 数据集：用warped proposal windows作为input，0.5 IoU 和0.3 IoU作为标准得到input的target
- 任务：**逐类独立二分类**
- 拆掉微调分类头，固定骨干CNN
- 目的：训练特定类别的SVM
