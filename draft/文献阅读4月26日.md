### 1.变化检测CD的概念
1. 变化检测CD：变化检测的目标是通过对比同一区域不同时相的配准图像，为区域内每个像素分配二分类标签（即变化或未变化）
	![[Pasted image 20260425133635.png]]
	![[Pasted image 20260425133648.png]]
	![[Pasted image 20260425133700.png]]
	口语：给两张不同时间拍的照片，然后生成一张相同宽高的照片，这个照片的每个位置像素取0和1，0代表经过这么长时间了，图片在该位置没变化，1就是有变化。
2. 细节：变化的定义是什么？
- 不固定：变化的定义随应用场景而异，例如城市扩张 [2]、森林砍伐 [3]、灾害评估 [4] 等。
- LEVIR-CD：building CD dataset，判断的是建筑变化
- ![[Pasted image 20260425134508.png]]
- ![[Pasted image 20260425134521.png]]
- 因为是建筑，所以，道路不属于变化，游泳池也不属于，所目标结果是：
- ![[Pasted image 20260425134603.png]]
### 2. 难点
- 场景呈现的对象有很多：
	- 某些目标的外观与“感兴趣的变化”相似，容易导致误检
	- ![[Pasted image 20260425135516.png]]
	- 这个和建筑很像，容易判定为建筑，出现了FP

	- 物体在不同时间显示出不同的光谱特征“

### 3. 对模型的需求
- 识别场景中目标变化的高层语义信息
- 从复杂的无关变化中区分出真实变化
## 算法
### 1.整体
步骤：BIT-based model  
![[Pasted image 20260425140347.png]]
- 用卷积模块从每一张图像中分别获取特征图
- 将特征图输入到BIT中，得到增强的双时相特征
- 最后增强的双时相特征输入预测头生成像素级预测结果
进一步理解：BIT
- semantic tokenizer：
	- 动机：输入图像中的感兴趣的变化能被少量高级概念（语义token）描述，这些概念能被双时图像共享。
		- 个人理解：例如有变化检测，以建筑变化为目标，那么semantic tokenizer就能够学会提取常用于对比建筑变化的特征。
	- 结果：每张图产生一个语义token集合，这个集合中有L个embedding size为C的语义token
- Transformer Encoder
	- 动机：把两个集合混合成一个，充分挖掘语义token之间的全局语义关联，为每一个时相图产生上下文丰富的token表示。
		- 个人理解：语义tokenizer是基于CNN的，受到感受野的限制，每个token只包含单时相局部的特征，通过transformer encoder后，充分挖掘了token之间的语义关联，时间和空间特征的关联都得到了挖掘。比如，一个白色的屋顶--> 草地旁边的白色屋顶，以后会成黑色。
	- 结果：一个上下文丰富的语义token集合，再分开，前一半给上面那个图像，下一半给下面的图像。
- Transformer Decoder
	- 动机：挖掘X^i的每个像素点与T中每个语义token之间的关联，让每个像素用语义token的组合来表示
	- 结果：得到优化特征X^i_new
### 2. Semantic Tokenizer 
![[Pasted image 20260425152601.png]]
- ![[Pasted image 20260425152721.png]]
- 步骤一：用L个`3*3`卷积核对图像特征作卷积，得到L个(H, W)的特征图
- 步骤二：对每个特征图作softmax，得到L个注意力图
- 步骤三：第i个注意力图与第1张特征图开始作逐元素相乘再相加的操作，就能得到c个数字，这一串数字就是第i个语义token。
	- 具体到代码实现，就是沿着空间维度展平，再矩阵相乘
### 3. Transformer Encoder
- ![[Pasted image 20260425153822.png]]
- 结构：由N_E层多头自注意力和MLP模块组成
- 输入：合并两个时相的一个语义token集合
- 输出：一个上下文丰富的语义token集合
- 细节：
	- 采用`前归一化残差单元`：more stable and competent
	- MLP中，使用的激活函数是GELU
	- 存在位置嵌入：个人感觉T1和T2内部没有位置编码也没关系，但是T1与T2之间应该得有位置编码，因为时间有先后。
	- 多少个注意力头，每个注意力头的子空间维度，MLP两层之间的feature size，就没必要讲了。
### 4. Transformer Decoder
![[Pasted image 20260425155347.png]]
- 结构：由N_D层多头交叉注意力和MLP模块组成
- 输入(以左边为例)：与第一个时相图关联的特征X1和语义token集合T^1_new
- 输入：优化特征X^i_new
- 操作细节：
	- 对X1在空间维度展平，得到`(H*W,C)`的矩阵，因此，每个方块代表一个像素，每个像素向量长度为c
	- 像素作为q，语义token作为kv，作交叉注意力，实现每个像素用语义token的组合来表示
- 结构细节：
	- 去除了位置嵌入
	- 去除了掩码多头自注意力模块
		- 说明：不在乎像素点之间的关联，只在乎与像素与语义对象之间的关联

## 结论
- BIT-based model 高效且有效地能够处理遥感图像的CD问题，打败了其他基于CNN和基于注意力的方法，达到了SOTA。
- 不是对pixel两两之间的关系进行建模，而是用BIT将图像转化为语义向量，然后对像素与语义向量之间进行建模。
- 文章提出的对CNN特征图的特征增强方案，具有很好的通用性。
